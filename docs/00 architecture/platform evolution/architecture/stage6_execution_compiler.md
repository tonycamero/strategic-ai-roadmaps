# Stage-6 Execution Compiler

The Stage-6 Execution Compiler transforms a Selection Envelope into deterministic SOP tickets.

This compiler is part of the StrategicAI Authority Spine.

Its purpose is to convert accepted strategic proposals into reproducible execution artifacts.

Stage-6 must be fully deterministic.

Given the same envelope and inventory registry, the compiler must always produce the same set of SOP tickets.

---

# Inputs

The compiler consumes the following artifacts.

selection_envelopes

The canonical execution intent generated by SAS elections.

selection_envelope_items

Each item represents an accepted proposal mapped to a capability.

inventory_registry

Defines all valid capabilities and their dependency metadata.

sas_runs

Provides the projectionHash that binds tickets to the diagnostic state.

---

# Output

The compiler produces execution artifacts in:

sop_tickets

Each ticket represents an executable operational step derived from a capability.

---

# Deterministic Identity

Each ticket must have a stable deterministic identity.

ticketKey is derived from:

selectionEnvelopeId
capabilityId
capabilityNamespace

Example rule:

ticketKey = hash(
  selectionEnvelopeId +
  capabilityId +
  capabilityNamespace
)

The ticketKey guarantees that recompiling the same envelope will update existing tickets rather than creating duplicates.

---

# Compiler Steps

The compiler runs the following sequence.

1 Load Selection Envelope

Load envelope and all selection_envelope_items.

2 Extract Capability Mapping

Extract capabilityId and capabilityNamespace from the proposal sourceAnchors JSONB.

3 Validate Capability

Verify capabilityId exists in the inventory registry.

Verify namespace compatibility.

Verify dependency metadata exists.

The compiler must never infer capabilities.

4 Resolve Capability Dependencies

Load dependency metadata from the inventory registry.

Expand dependent capabilities if required.

5 Generate Ticket Identity

Compute deterministic ticketKey.

6 Upsert SOP Ticket

Insert or update the ticket using:

UPSERT ON CONFLICT(ticketKey)

7 Bind Provenance

Each ticket must record provenance fields.

selectionEnvelopeId
sourceFindingIds
envelopeVersion
generationEventId
projectionHash

These fields ensure tickets can be traced back to their originating diagnostics.

---

# Provenance Fields

Each sop_ticket contains:

ticketKey
selectionEnvelopeId
sourceFindingIds
envelopeVersion
generationEventId
projectionHash

These fields allow full reconstruction of how a ticket was generated.

---

# Determinism Rules

The Stage-6 compiler must obey the following invariants.

No LLM capability inference.

Capabilities must exist in the inventory registry.

ticketKey must be deterministic.

Recompilation must not create duplicate tickets.

Compiler output must depend only on:

selection envelope
inventory registry
projection state

---

# Result

Stage-6 converts strategy into operational intent.

selection_envelope
    ↓
Stage-6 Execution Compiler
    ↓
sop_tickets

These tickets become the input for the Stage-7 Roadmap Graph Compiler.
